API DOCUMENTATION

POST/ register

Items needed for user registration

Property Type Description
name string username
email string user email
password string user password
idioms string languages ​​the user knows
profession string user's profession
located string user location
tel1 string first user phone
tel2 string second user phone
about string about the user
images array of string profile pictures
image_banner string profile background photo
image_profile string user photo

Registry response examples

Property Type Description Json
error string Error message indicating that the user already exists 
  {
    "error": "User already exist"
  }
register boolean Indicates whether the registration was successful 
  {
    "register": true
  }

GET/ login

Items needed for user login

Parameters Type Description
email string Email of user trying to authenticate
password string Password of the user trying to authenticate

Login response examples

HTTP Status Code Response Body Description
200 
  {
    "user_id":4,
    "name":"Pedro Moura",
    "email":"pflmoura@gmail.com",
    "idioms":"Portugues, Inglês",
    "profession":"Estudante",
    "locate": "Rio de janeiro, Brasil",
    "tel1":"+5521987461245"
    "tel2":"+5521972340243"
    "about": "Meu nome é Pedro"
    "images": [
    "http://www.daniellenoce.com.br/wp-content/uploads/2017/07/9-vistas-incriveis-no-rio-de-janeiro-danielle-noce-9.jpg",
    "https://www.daninoce.com.br/wp-content/uploads/2017/07/9-vistas-incriveis-no-rio-de-janeiro-danielle-noce-02.jpg"
    ],
    "image_banner": "https://www.daninoce.com.br/wp-content/uploads/2017/07/9-vistas-incriveis-no-rio-de-janeiro-danielle-noce-02.jpg",
    "image_profile": "https://www.shutterstock.com/shutterstock/photos/492607711/display_1500/stock-photo-funny-nerd-or-geek-have-an-idea-and-holds-finger-up-492607711.jpg"
  }
Login successful. The response body is an object containing the user's information, excluding their password. The auth-token header is also included in the response, containing a JWT token that can be used to authenticate future requests.
400 
  {
     error: "User or password wrong" 
  }
Invalid email or password provided. The response body is an object containing an error property with the value "User or password wrong".
401 
  {
    error: "Authentication token is missing or invalid" 
  }
The request did not include a valid JWT token in the auth-token header, or the token provided was invalid. The response body is an object containing an error property with the value "Authentication token is missing or invalid".
